点击顶栏的齿轮图标打开**设置**。设置侧边栏共 9 个分类,顺序为:通用、外观、下载、BitTorrent、eD2K、代理、API 服务、扩展、关于。本页按分类逐项说明每个设置项及其默认值。「扩展」分类以两个 Tab 承载插件管理与可选外部组件(FFmpeg、yt-dlp),详见[插件文档](/docs/zh/plugins/overview/)。设置侧边栏顶部的搜索框可以按名称直接定位到任意设置项。

## 通用

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 开机自启动 | 系统启动时自动运行 FluxDown。 | 关 |
| 关闭时最小化到托盘 | 点击关闭按钮时隐藏到系统托盘,而非直接退出应用。 | 开 |
| 桌面悬浮球 | 桌面常驻置顶小组件,显示下载速度与进度,可将链接或种子文件拖到它上面创建任务。Wayland 环境下不可用(见下方说明)。 | 关 |
| 剪贴板监听(仅 Linux Wayland) | 主窗口隐藏时监测剪贴板中的下载链接并通知。仅当悬浮球在 Wayland 下不可用时才会显示,作为替代方案。 | 关 |
| 关联 .torrent 文件 | 将 FluxDown 设为 `.torrent` 文件的默认打开方式。若尚未设置,首次启动时会弹窗询问一次。 | 关 |
| 完成通知 | 任务完成时弹出系统通知。 | 开 |
| 下载时保持唤醒 | 有任务下载时阻止系统睡眠或息屏,任务全部结束后自动恢复。 | 关 |
| 侧边栏显示(状态/队列/分类) | 选择侧边栏中显示哪些区块。三项全部关闭时侧边栏整体隐藏。 | 全部开 |
| 标题栏按钮(全部暂停/全部恢复/设置/主题) | 选择顶栏显示哪些工具按钮,也可直接右键某个按钮单独隐藏。 | 全部开 |
| 自定义分类 | 为侧边栏的"分类"区块新增文件类型筛选,按扩展名或正则表达式匹配。 | 无 |

## 下载

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 默认保存目录 | 新建下载任务时的默认保存位置,除非手动选择其他目录或队列有自己的设置。 | 系统默认下载目录 |
| 跟随上次保存位置 | 使用上次下载的目录作为下一次的默认目录,而非固定的默认保存目录。 | 关 |
| 免打扰下载 | 外部下载请求(如浏览器扩展发起的)跳过确认框,直接按默认设置立即开始下载。 | 关 |
| 默认线程数 | 新建下载任务的默认分片数量(0 = 根据文件大小和 CPU 核心数自动计算)。 | 自动 |
| 最大同时下载数 | 所有未单独设置该项的队列共享的最大同时下载任务数。 | 5 |
| 速度限制 | 全局下载速度上限,单位 KB/s(0 = 不限制)。也可在状态栏中直接调整。 | 0(不限制) |
| 失败重试次数 | 网络中断等瞬时错误后自动重试的次数(-1 = 无限,0 = 关闭)。 | 3 |
| 重试间隔 | 每次自动重试前的等待秒数。 | 5 |
| User-Agent | 下载请求使用的浏览器标识;内置 Chrome/Firefox/Edge/Safari 预设。留空使用内置 Chrome UA。 | 空(内置 Chrome UA) |
| 文件管理器命令 | 在第三方管理器中打开文件或目录的自定义命令模板。占位符 `{path}`(当前路径)、`{dir}`(目录)。留空使用平台默认(资源管理器/Finder/Nautilus)。 | 空 |
| 默认下载队列 | 未指定队列时新任务归属的队列(浏览器扩展下载也使用此队列)。 | 内置主队列 |

### 第三方文件管理器

"在文件夹中显示"与"打开目录"会跟随系统的**默认文件管理器**,大多数情况下无需任何配置:

- **Windows**:遵循 `Directory\shell` 下注册的默认文件夹处理程序(即双击文件夹时打开的程序)。被设为默认的第三方管理器会被自动使用——**OneCommander 即为开箱即用**;Directory Opus、Total Commander、Files 等将自身注册为默认文件夹处理程序的管理器也以同样方式被识别。由于 Windows 只允许资源管理器选中具体文件,当第三方管理器为默认时,"在文件夹中显示"会打开所在文件夹,而非高亮该文件。
- **macOS**:使用 Launch Services 中 `public.folder` 的默认处理程序。
- **Linux**:使用 `mimeapps.list` 中 `inode/directory` 的默认程序(经 `xdg-open`)。

若要使用**未**设为系统默认的管理器,或希望在第三方管理器中精确高亮文件,请填写上方的**文件管理器命令**模板。一条命令同时用于"在文件夹中显示"(下载好的文件)和"打开目录"(目录),FluxDown 会按场景自动填充占位符。占位符**已自动做 shell 转义,请勿再自行加引号**——但可执行文件路径若含空格,仍需自己加引号:

- `{path}`:当前路径。显示文件时为完整文件路径,打开目录时为目录路径。
- `{dir}`:目录。显示文件时为文件所在父目录,打开目录时为该目录本身。

Windows 命令示例(请按实际安装路径调整;能定位到具体文件的管理器用 `{path}`,只能打开目录的用 `{dir}`):

| 管理器 | 命令 |
|---|---|
| OneCommander | `"C:\Program Files\OneCommander\OneCommander.exe" {path}` |
| Everything | `"C:\Program Files\Everything\Everything.exe" -select {path}` |
| Directory Opus | `"C:\Program Files\GPSoftware\Directory Opus\dopusrt.exe" /cmd Go {path} NEW` |
| Total Commander | `"C:\totalcmd\TOTALCMD64.EXE" /O /T {dir}` |

已配置的模板始终优先于自动识别;若模板启动失败,FluxDown 会回退到平台默认方式。

## BitTorrent

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 监听端口范围 | 接收 BT 连接使用的端口范围。 | 6881–6891 |
| Tracker 列表 | 在 25 个内置 Tracker(亚洲优先排序)之外,自定义用于发现对等节点的 Tracker 服务器,每行一个,可一键恢复为内置列表。 | 25 个内置 Tracker |
| Tracker 订阅 | 定期从社区维护的订阅源获取最新 Tracker,与上方列表自动合并去重。 | 开 |

部分 BT 设置需要重启 BT 引擎才能生效(该分类下会显示提示)。DHT 与 UPnP 在引擎内部始终开启,没有单独的开关。下方的代理设置不适用于 BT 下载。

## eD2K

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 服务器列表 | 手动添加的 eD2K 服务器(`host:port`,每行一个),用于找源,与订阅列表自动合并去重。 | 空 |
| 服务器订阅 | 定期从社区维护的 `server.met` 获取最新服务器列表,与上方列表自动合并去重。 | 开 |
| Kad DHT 找源 | 通过去中心化的 Kad 网络找源,服务器全部失联时仍可用。 | 开 |
| UPnP 端口映射 | 通过 UPnP 自动映射端口以获得 HighID,提升连通性。 | 开 |
| 监听端口 | eD2K 客户端使用的 TCP/UDP 端口(0 = 由系统自动选择)。 | 0(自动) |

## 代理

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 模式 | 不使用代理、系统代理(读取系统设置)或手动配置。 | 不使用代理 |
| 类型(手动模式) | HTTP、HTTPS、SOCKS4 或 SOCKS5。 | HTTP |
| 服务器地址/端口(手动模式) | 代理服务器的地址与端口。 | 空 |
| 用户名/密码(手动模式) | 可选的代理认证信息。 | 空 |
| 排除列表 | 不经过代理、直接连接的地址,逗号分隔。 | 空 |
| 测试连接 | 校验当前代理配置,返回延迟或错误信息。 | —— |

代理设置仅对 HTTP/HTTPS/FTP/eD2K 下载生效——BT 下载始终直连,不经过代理。

## API 服务

一个仅监听本机(127.0.0.1)的 HTTP API,供浏览器扩展、aria2 兼容工具与自动化脚本调用。

| 设置项 | 作用 | 默认值 |
|---|---|---|
| 启用 API 服务 | 本机 HTTP 服务总开关,下方三个功能开关都依赖它开启才能生效。 | 开 |
| 监听端口 | 本机服务监听的端口(1024–65535)。 | 17800 |
| 访问令牌 | 用于校验 API 请求的令牌,字段旁提供生成/复制按钮。启用管理 API 后强制要求非空。 | 空 |
| 浏览器脚本接管 | 供 FluxDown 油猴脚本在 `http://127.0.0.1:<端口>` 接管浏览器下载,附带复制脚本按钮。 | 开 |
| aria2 RPC 兼容 | 在 `/jsonrpc` 实现 aria2 JSON-RPC 协议(`addUri`/`getVersion`/`getGlobalStat`/`multicall` 等),供"发送到 aria2"类脚本或 AriaNg 等客户端使用。 | 开 |
| 管理 API | 在 `/api/v1` 提供任务查询与控制的 HTTP API,供 MCP、自动化脚本等外部程序调用,始终强制要求访问令牌。 | 关 |

完整接口说明见 [API 文档](/api-docs)。
