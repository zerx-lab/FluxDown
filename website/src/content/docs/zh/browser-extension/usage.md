## 下载拦截的工作方式

当**下载拦截**开启时(默认开启),只要你点击的内容被扩展识别为可下载文件,FluxDown 就会在浏览器自身的下载界面出现之前介入——原生的"另存为"弹窗和下载栏都不会出现——并通过[安装扩展](/docs/zh/browser-extension/install/)中介绍的本地连接,把请求转交给 FluxDown 桌面客户端。默认情况下,桌面客户端随后会弹出自己的快速下载确认框,让你在开始传输前确认文件名、保存目录和线程数。无论下载是由普通链接、JavaScript 跳转还是表单提交触发的,这套流程都一样有效——不管页面用什么方式触发下载,扩展都能捕获到。

如果尝试发送下载的那一刻桌面客户端不可达,扩展会自动放行,让浏览器像平常一样正常下载该文件,而不是卡住不放,同时弹出一条"未检测到 FluxDown 应用"的提示。这条提示在约 30 秒内不会因为后续下载而重复弹出,一旦客户端恢复响应,扩展也会静默恢复接管。

## 选择拦截模式

打开弹窗 → **快捷设置** → **拦截模式**:

| 模式 | 弹窗中的名称 | 行为 |
| --- | --- | --- |
| `smart` | 智能模式*(默认)* | 综合文件扩展名、MIME 类型与文件大小判断。如果浏览器本身已经把它当作下载处理,而它看起来又不像普通网页资源(HTML/CSS/JS、页面内小图片),就会兜底拦截。 |
| `extension` | 仅扩展名 | 只在文件名或 URL 以下方配置的某个扩展名结尾时才拦截,不考虑其他因素。 |
| `all` | 拦截所有 | 拦截除排除域名之外的所有下载。 |

无论选择哪种模式,都会先经过两道统一判断:排除域名中的下载直接放行,小于**最小文件大小**的文件也交给浏览器处理。

## 精细调整拦截范围

- **最小文件大小**(快捷设置):可选`不限`、`100 KB`、`512 KB`、`1 MB`、`5 MB`、`10 MB`。小于此大小的下载交给浏览器处理,适合把图标、小片段之类的文件排除在 FluxDown 之外。该选项只在浏览器能报告文件大小时生效;大小未知的下载仍按正常逻辑判断。
- **拦截文件类型**:扩展预置了大约三十多个扩展名——压缩包、安装程序、磁盘镜像、视频、音频、办公文档,以及 `.apk`、`.ipa`、`.torrent`。点击 **+** 按钮添加(输入 `pdf` 或 `.pdf` 都可以),点击标签上的 **×** 移除。这份列表只在"仅扩展名"模式下起决定性作用;在智能模式下它只是多个判断依据之一。
- **排除域名**:列表中域名的下载始终交给浏览器,与拦截模式无关。点击 **+** 手动输入域名,或点击**当前站点**一键排除当前标签页所在的域名。点击域名旁的 **×** 可以移除。

## 开启或关闭拦截

弹窗顶部的**下载拦截**开关是总开关。关闭后,所有下载都会回到浏览器的正常流程,包括原本会被拦截屏蔽掉的那个确认弹窗。

不必打开弹窗也能切换:在浏览器任意页面按下 **Alt+Shift+D** 即可立即切换拦截状态,并有一条系统通知确认结果。这是用浏览器自带下载器抓取单个文件的最快方式——关掉拦截、点击链接、再打开拦截——比如遇到 FluxDown 对某个链接判断有误的站点时很好用。(如果 Alt+Shift+D 与其他功能冲突,可以到浏览器的扩展快捷键设置页面重新绑定。)如果是某个站点永远不想被拦截,更适合直接加入**排除域名**——一劳永逸,不用每次都记得手动切回来。

工具栏图标本身也会反映当前状态(拦截关闭时图标会变暗),不用打开弹窗也能一眼看出。

## fluxdown:// 协议模式

通常情况下,扩展经 Native Messaging 把下载转交给 FluxDown 客户端。弹窗快捷设置和选项页中的 **FluxDown 自定义协议**开关(默认关闭)会切换到另一条通道:被拦截的下载导航到一个 `fluxdown://download?url=...&filename=...` 协议 URL,由操作系统唤起注册了该协议的应用——FluxDown 桌面或 Android 客户端。

适用场景因平台而异:

- **Android**——这是*唯一*可用通道。支持扩展的 Android 浏览器(Edge、Firefox、Kiwi 及其他 Chromium 分支)完全没有 Native Messaging,拦截要送达应用就必须打开此开关。协议 URL 触发系统 VIEW intent,唤起 FluxDown Android 应用并打开新建下载弹层,URL(以及页面提供的建议文件名)已经预填好;你在弹层里确认目录和线程数后开始下载。批量下载(比如在资源面板中多选)会逐条送达、间隔片刻,并作为新行合入同一个已打开的弹层——一次批量最终汇成一张填好所有 URL 的表单。
- **桌面端(Windows、macOS、Linux)**——此开关同样生效:开启后下载不再走 Native Messaging,而是经系统协议处理器送达客户端,进入同一条外部下载流程(快速下载确认框;开了免打扰下载则静默建任务)。Native Messaging 仍是更好的通道——它能携带协议模式带不了的 Cookie、请求头、请求方法和请求体——所以桌面端建议保持关闭,除非 Native Messaging 不可用,比如浏览器策略禁用了它,或便携部署从未注册过本机主机。

两条限制在所有平台上都适用:

- **链接不携带任何凭据。**协议 URL 只带地址和文件名提示——Cookie、`Authorization` 请求头、请求方法和请求体都留在浏览器里。需要登录态的下载会得到错误页或权限拒绝;协议模式适用于可公开访问的文件。
- **音视频分轨下载会回退浏览器。**协议 URL 只能表达一个地址,嗅探到的需要单独合并音频轨的媒体无法经此转交——与其下出一个无声视频,扩展会放行让浏览器正常下载。

不言而喻,设备上必须先装好 [FluxDown 客户端](/#download)——否则没有任何应用注册响应 `fluxdown://` URL,这次导航会无声无息地失败。

## 通过右键菜单发送链接、图片与媒体到 FluxDown

右键菜单中的 FluxDown 选项独立于下载拦截开关——即使拦截已关闭,这些选项依然可用:

| 右键点击的对象 | 菜单项 |
| --- | --- |
| 链接 | 使用 FluxDown 下载此链接 |
| 图片 | 使用 FluxDown 下载此图片 |
| 视频或音频元素 | 使用 FluxDown 下载此视频/音频 |
| 页面空白处 | 使用 FluxDown 下载此页面 |

最后一项会把当前页面自身的 URL 发送给 FluxDown——适合浏览器打开的"页面"其实就是一个原始文件的情况,比如直接在标签页里播放的视频或查看的 PDF。

## 检测页面中的媒体与可下载资源

FluxDown 会持续扫描每个页面中的可下载资源——视频、音频元素,流媒体清单(HLS 的 `.m3u8`、DASH 的 `.mpd`),字幕文件,以及看起来像文件的链接或网络响应——扫描 DOM 结构的同时,也实时监听页面的网络请求。检测到的资源会被分类(视频、音频、文档、压缩包、图片、种子、流媒体、字幕、磁力链接、其他)并做轻量过滤,让列表尽量只保留真正可下载的内容,而不是把每个图标、广告请求或预加载片段都塞进来:体积过小的文件按类型分别过滤掉,追踪/统计类请求会被忽略,原始的流媒体分片除非大到足以单独构成一个完整文件,否则也不会显示。图片嗅探默认关闭,因此页面上的普通图片不会出现在结果里。

页面内有两处 UI 展示检测结果:

- **悬浮球**——一个可拖拽的小圆点,默认停靠在页面左侧或右侧边缘(拖动后松手会自动吸附到最近的一侧),上面的徽标显示检测到的资源数量。点击它打开资源面板。如果不想看到它,可以在弹窗的**悬浮球**开关中隐藏,也可以直接点击资源面板头部的隐藏(眼睛)图标——隐藏悬浮球不影响资源检测,工具栏徽标也照常更新。
- **资源面板**——在悬浮球旁边展开,把检测到的资源按标签页分组(全部、视频、音频、文档、压缩包、流媒体、字幕、磁力、其他),并显示每类数量。勾选单个条目(或使用**全选**),点击**批量下载**按钮即可把所有勾选项一次性发送给 FluxDown;也可以用某一行自己的小**下载**按钮只下载那一项。

把鼠标悬停在视频元素上,还会弹出一个小的悬浮下载按钮,专门用于下载这一个视频,不需要打开资源面板。

扩展的工具栏图标会用数字徽标显示当前标签页检测到的资源数量,不用打开任何界面就能知道这个页面上有没有可下载的内容。

### 安装 FFmpeg 组件以合并输出

你嗅探到的资源里,有不少——尤其是流媒体清单——并不会以单个成品文件的形式到手。DASH 流(`.mpd`)通常会把音频和视频拆成两条独立轨道,因此下载后会得到两个文件(例如 `video.mp4` 和 `video.audio.m4a`),除非 FluxDown 能把它们合并。这一合并由 **FFmpeg** 完成(纯流复制,不重新编码,速度很快),它不随应用打包分发——而是一个可选组件,需要你在桌面客户端的**设置 → 扩展 → 组件**页按需安装。装好后,DASH 的音视频轨会自动合并成一个可直接播放的文件;不装则两条轨道保留为独立文件,需要你自己合并。安装 FFmpeg 还会一并解锁声明了 `ffmpeg` 能力的插件(见[插件 API 参考](/docs/zh/plugins/api-reference/)),并同时安装 `ffprobe`。HLS(`.m3u8`)下载不需要 FFmpeg——FluxDown 会自行把它们重组并 remux 成 MP4。

## 查看今日统计

弹窗的**今日统计**区域显示当天的两个计数:**已接管**(成功转交给 FluxDown 的下载数)与**失败**(未能成功转交的次数,通常是因为当时连不上桌面客户端)。两个计数都会在每天开始时自动清零,也可以点击弹窗底部的**重置统计**立即清零。

## 切换主题与语言

- **主题**——头部的太阳/月亮图标用于在浅色与深色之间切换弹窗外观。在你点击它之前,弹窗会自动跟随系统主题。
- **语言**——头部的语言按钮(显示 `EN` 或 `中`)会同时切换弹窗界面和页面内悬浮球/资源面板的语言,在中英文之间切换。FluxDown 首次运行时会根据浏览器语言自动选择,之后手动切换会覆盖这个默认值并被记住。
